# Secure code review

- [OWASP Code Review Guide](https://owasp.org/www-project-code-review-guide/) — `[L1 Foundation]` `[Required]` · Module 6 · Official project. A structured manual review approach. It matters for communicating behavior, preconditions, impact, remediation, regression tests, and telemetry.
- [OWASP Top 10](https://owasp.org/www-project-top-ten/) — `[L3 Integrated]` `[Required]` · Module 6 · Official project. Common application-security risk categories. It matters when the full exercise set expands beyond traffic-specific reliability issues.
- [OWASP API Security](https://owasp.org/www-project-api-security/) — `[L3 Integrated]` `[Required]` · Modules 2 and 6 · Official project. Authorization and resource-consumption risks in APIs. It matters for reviewing synthetic workflow endpoints.
- [Semgrep rules](https://github.com/semgrep/semgrep-rules) — `[L2 Applied]` `[Optional]` · Module 6 · Project repository. Inspectable static-analysis patterns. It matters for learning pattern design and limits; findings still require reachability and impact analysis.
- [PortSwigger Web Security Academy](https://portswigger.net/web-security) — `[L3 Integrated]` `[Optional]` · Module 6 · Secondary training reference. Selected vulnerability explanations and exercises. It matters for targeted gaps, not as a required complete course.
