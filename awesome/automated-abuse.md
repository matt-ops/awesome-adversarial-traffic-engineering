# Automated abuse

- [OWASP Automated Threats to Web Applications](https://owasp.org/www-project-automated-threats-to-web-applications/) — `[L1 Foundation]` `[Required]` · Module 2 · Official project. A workflow-oriented automated-threat taxonomy. It matters because it separates attacker outcomes such as credential stuffing, scalping, and inventory denial instead of treating all bots alike.
- [OWASP Web Security Testing Guide](https://owasp.org/www-project-web-security-testing-guide/) — `[L2 Applied]` `[Required]` · Module 2 · Official project. Practical web test methodology and domain references. It matters for connecting abuse hypotheses to authorized, reproducible test cases.
- [OWASP API Security Project](https://owasp.org/www-project-api-security/) — `[L2 Applied]` `[Recommended]` · Modules 2 and 6 · Official project. API-specific authorization, resource-consumption, and workflow risks. It matters because abusive traffic often targets stateful APIs rather than pages.
- [PortSwigger Web Security Academy](https://portswigger.net/web-security) — `[L3 Integrated]` `[Optional]` · Modules 2 and 6 · Secondary training reference. Selected business-logic and race-condition material provides deeper application context. It matters when a capstone hypothesis depends on a particular server-side behavior; the full curriculum is not required.
