# JavaScript runtime research

- [MDN JavaScript Guide](https://developer.mozilla.org/en-US/docs/Web/JavaScript/Guide) — `[L2 Applied]` `[Recommended]` · Modules 3–4 · Official reference. Covers language fundamentals needed to trace local sensor code. It matters before runtime instrumentation.
- [Chrome DevTools JavaScript debugging](https://developer.chrome.com/docs/devtools/javascript/) — `[L3 Integrated]` `[Required]` · Modules 3–4 · Official documentation. Breakpoints, scopes, source maps, and runtime inspection. It matters for following a signal from collection through transformation.
- [Chrome DevTools Protocol Runtime domain](https://chromedevtools.github.io/devtools-protocol/tot/Runtime/) — `[L3 Integrated]` `[Recommended]` · Module 3 · Official protocol documentation. Describes execution contexts, evaluation, console, and exception events. It matters for cross-context observation.
- [Chromium source](https://source.chromium.org/chromium) — `[L4 Deep]` `[Optional]` · Modules 1 and 3 · Official source. Supports narrow implementation questions after version-pinned experiments. It matters for resolving browser behavior, not for Foundation study.
