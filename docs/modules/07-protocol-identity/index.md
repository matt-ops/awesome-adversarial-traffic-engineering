# Module 07 - Protocol identity

**Outcome:** compare claimed browser identity with TLS and HTTP behavior, explain
what intermediaries can change, and avoid treating fingerprints as identity proof.

| Depth | Lessons and evidence |
|---|---|
| Foundation | ClientHello purpose and fingerprint-as-pivot model |
| Applied | Local configuration/HTTP comparison and JA4/JA4H interpretation |
| Integrated | HTTP/2 streams, connection reuse, and intermediary boundaries |
| Deep | HTTP/3 and QUIC identity/migration limits |

Start with [TLS ClientHello](01-tls-clienthello.md).
