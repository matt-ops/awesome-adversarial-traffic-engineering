# Module 04 - Automated abuse and workflow attacks

**Outcome:** translate a business-abuse objective into an entry-point and state
map, establish a blocked baseline, and prove whether a controlled hostile
workflow completes its protected action.

| Depth | Lessons and evidence |
|---|---|
| Foundation | Abuse objectives, API/workflow mapping, authentication and rate controls |
| Applied | Local inventory authorization flaw and provider-assigned authentication lab |
| Integrated | Inventory/promotion chain plus provider-assigned limit-overrun race |
| Deep | Remediation, regression test, and exact replay of the original workflow |

Start with [automated-abuse objectives](01-abuse-objectives.md). Docker appears
only after the HTTP, browser, JavaScript, and Playwright prerequisites.
