# Module 03 - Playwright foundations

**Outcome:** build and explain a deterministic local browser workflow, preserve
DOM and network evidence, and reason about context/state isolation before using
automation as an adversarial capability.

## Foundation

Complete [the object model](01-object-model.md), [first local
browser](02-first-browser.md), [contexts and state](03-contexts-and-state.md),
and [network events](04-network-events.md). Produce the headed workflow and trace.

## Applied

Complete [frames, workers, and CDP](05-frames-workers-and-cdp.md). Correlate one
top-page action with frame, worker, browser, and network evidence.

## Integrated

Reuse the instrumented workflow to compare manual, headed, headless, and
HTTP-client populations while keeping target action and reset state fixed.

## Deep

Reuse context isolation, storage state, frame/worker boundaries, and protocol
observations in identity-coherence and browser-version drift work.

Start with the [Playwright object model](01-object-model.md). The first workflow
uses the static loopback site and does not use Docker.
