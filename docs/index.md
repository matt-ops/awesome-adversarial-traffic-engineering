# Adversarial Traffic Engineering

Build automated adversaries, test what traffic controls appear to rely on, prove
whether protected actions or service objectives can be defeated, and retest the
same attack after remediation.

This is one ordered, source-backed path for experienced network and security
practitioners entering browser automation, bot-control evasion, and bounded DDoS
red-team work.

The required path starts with HTTP and the edge. Formal red-team method and
engagement practice remain available in an [optional appendix](modules/00-method/index.md).

[Start the path](start-here.md){ .md-button .md-button--primary }

Do not install the full lab stack yet. The path tells you what to install at the
lesson where it becomes necessary.
