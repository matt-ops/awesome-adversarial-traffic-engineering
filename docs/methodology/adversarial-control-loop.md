# The adversarial-control testing loop

!!! note "Course synthesis"
    **COURSE_SYNTHESIS:** AATE joins established assessment, emulation, mapping,
    and experimental principles into the following specialized loop. No cited
    source publishes this exact sequence.

```text
authorize -> objective -> map -> legitimate baseline -> simplest adversary
-> blocked baseline -> candidate relied-upon signals -> falsifiable hypothesis
-> controlled change -> repeat action -> prove outcome -> residual anomalies
-> impact/limits -> remediation criteria -> identical retest
```

## The fifteen steps

1. Authorize and define scope.
2. Define the adversary objective and protected action.
3. Map the application, workflow, control, and resource path.
4. Record the legitimate baseline.
5. Build the simplest representative adversary.
6. Record the blocked or challenged baseline.
7. Enumerate candidate relied-upon signals and assumptions.
8. State a falsifiable bypass or pressure hypothesis.
9. Change one variable or one deliberately coherent signal set.
10. Repeat the same protected action.
11. Prove whether the adversarial objective succeeded.
12. Record residual anomalies and alternative explanations.
13. Explain impact and limitations.
14. Recommend remediation and measurable success criteria.
15. Repeat the same attack after remediation.

The [provenance page](provenance.md) identifies which parts come directly from
NIST, MITRE, OWASP, experiment design, and AATE specialization.
