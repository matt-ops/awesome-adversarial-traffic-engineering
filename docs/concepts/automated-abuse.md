# Automated abuse

Model abuse by outcome and workflow rather than by tool. Credential stuffing seeks account access; account creation builds identity supply; scraping extracts information; scalping acquires scarce goods; inventory denial holds state; promotion abuse captures subsidy; application-layer denial of service consumes expensive work.

For each case record actor, asset, goal, entry point, state transition, observable behavior, control, telemetry, legitimate lookalike, residual risk, and validation. Automation can be legitimate, and human-driven activity can be abusive.
