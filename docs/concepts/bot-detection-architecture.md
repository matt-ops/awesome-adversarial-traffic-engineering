# Bot-detection architecture

A layered system collects edge, protocol, browser, session, account, behavior, and workflow observations; derives explainable features; evaluates rules or scores; and chooses allow, observe, challenge, throttle, or block. Stateful evidence and feedback loops add power and operational risk.

Design for reason codes, population metrics, shadow rollout, privacy, latency, missing data, drift, rule ownership, appeals, and retesting. Red-team findings should produce a measurable defensive change rather than a one-time bypass narrative.
