# Browser process model

The browser process coordinates windows, profiles, networking, and child processes. Renderer processes execute page content; frames can use separate renderers; workers and service workers have different lifecycles and contexts. Playwright’s BrowserContext provides isolated browser-session state but is not a separate host.

CDP exposes targets, frames, execution contexts, network events, and runtime events. Name the observation layer before drawing a conclusion. Local version-pinned behavior can drift across browser releases.
