# Authorization and scope

Authorization must name the owner, target, environment, time window, objective, allowed techniques, traffic envelope, data handling, and stop authority. A service being reachable is not authorization.

For AATE, the target set is fixed to loopback addresses and approved Compose names. A client must validate the parsed hostname—not a string prefix—and reject credentials, arbitrary DNS names, public or private network addresses, redirects to external hosts, and excessive parameters.

Scope also includes non-target dependencies and people. Identify shared databases, queues, developer machines, monitoring owners, and cleanup obligations. Stop when observations no longer match the authorized model.

In an interview, lead with the business objective and authority, then environment, exclusions, caps, monitoring, abort, recovery, evidence, remediation, and retest.
